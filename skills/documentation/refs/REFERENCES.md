# Documentation — References

## Frameworks

- **Diátaxis** — The four-quadrant technical documentation framework (Tutorial / How-To / Explanation / Reference)
  https://diataxis.fr/

- **Architecture Decision Records (ADR)** — Michael Nygard's original proposal and tooling for ADRs
  https://adr.github.io/

## Changelogs

- **Keep a Changelog** — The community standard for CHANGELOG.md format and categories
  https://keepachangelog.com/

## API Documentation

- **OpenAPI 3.1.0 Specification** — The full OAS 3.1 specification
  https://spec.openapis.org/oas/v3.1.0

## Code Documentation Style

- **Google Python Style Guide — Comments and Docstrings** — Canonical Google docstring format for Python
  https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings

- **JSDoc** — The standard documentation tool for JavaScript and TypeScript
  https://jsdoc.app/

- *The Documentation System* (Canonical) — Diátaxis explained with worked examples — https://docs.divio.com/documentation-system/
- RFC 2119 — Key words for use in RFCs (MUST, SHOULD, MAY) — https://www.rfc-editor.org/rfc/rfc2119
- Conventional Commits (for CHANGELOG generation) — https://www.conventionalcommits.org/
