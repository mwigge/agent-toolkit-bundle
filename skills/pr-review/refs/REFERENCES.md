# PR/MR Review — References

## Google Engineering Practices

- **Google's Code Review Developer Guide** — Comprehensive guide covering what to look for, how to write review comments, and reviewer / author etiquette
  https://google.github.io/eng-practices/review/

## Commit Standards

- **Conventional Commits Specification** — Machine-readable commit message format that drives changelog generation and semantic versioning
  https://www.conventionalcommits.org/

## Security

- **OWASP Top Ten** — The ten most critical web application security risks; every reviewer should know these
  https://owasp.org/www-project-top-ten/

- **OWASP SQL Injection Prevention Cheat Sheet** — Parameterised queries, stored procedures, input validation
  https://cheatsheetseries.owasp.org/cheatsheets/SQL_Injection_Prevention_Cheat_Sheet.html

## GitLab

- **GitLab Merge Request Reviews** — GitLab-specific review features: suggestions, review batching, approval rules
  https://docs.gitlab.com/user/project/merge_requests/reviews/

## Additional Reading

- *Accelerate* — Nicole Forsgren, Jez Humble, Gene Kim — data on how code review practices affect delivery performance
- *A Philosophy of Software Design* — John Ousterhout — deep treatment of complexity, naming, and module design
- OWASP Code Review Guide — https://owasp.org/www-project-code-review-guide/
