# API Designer — Reference Links

## Specifications
- https://spec.openapis.org/oas/v3.1.0 — OpenAPI 3.1.0 specification (full normative text)
- https://json-schema.org/specification.html — JSON Schema (draft 2020-12): used for OpenAPI 3.1 schemas
- https://jsonapi.org/format/ — JSON:API 1.1: conventions for resource relationships, pagination, filtering

## REST Design Guidelines
- https://learn.microsoft.com/en-us/azure/architecture/best-practices/api-design — Microsoft REST API Design Guidelines: naming, versioning, pagination
- https://github.com/microsoft/api-guidelines/blob/vNext/azure/Guidelines.md — Microsoft Azure API Guidelines (detailed, authoritative)
- https://google.aip.dev/ — Google API Improvement Proposals: resource-oriented design, naming, error model

## Error Handling
- https://www.rfc-editor.org/rfc/rfc7807 — RFC 7807: Problem Details for HTTP APIs (type, title, status, detail, instance)
- https://www.rfc-editor.org/rfc/rfc9457 — RFC 9457: Problem Details for HTTP APIs (updates/supersedes RFC 7807)

## Linting & Validation
- https://stoplight.io/open-source/spectral — Spectral: OpenAPI/AsyncAPI linter with built-in and custom rulesets
- https://github.com/stoplightio/spectral/blob/develop/docs/reference/openapi-rules.md — Spectral built-in OpenAPI rules reference

## Tooling
- https://editor.swagger.io/ — Swagger Editor: in-browser OpenAPI editor and validator
- https://www.redocly.com/docs/cli/ — Redocly CLI: lint, bundle, and preview OpenAPI specs
