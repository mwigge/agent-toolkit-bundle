# TDD Workflow — Reference Links

## Foundational Books
- https://www.amazon.com/Test-Driven-Development-Kent-Beck/dp/0321146530 — TDD by Example (Kent Beck): the definitive TDD text, with worked examples in Java and Python
- https://www.amazon.com/Growing-Object-Oriented-Software-Guided-Tests/dp/0321503627 — Growing Object-Oriented Software, Guided by Tests (Freeman & Pryce): outside-in TDD, acceptance + unit tests
- https://refactoring.com/ — Refactoring (Martin Fowler): catalogue of refactoring patterns for the Green→Refactor phase

## Core Principles
- https://martinfowler.com/bliki/TestDrivenDevelopment.html — Martin Fowler's TDD overview: Red-Green-Refactor cycle explanation
- https://qualitycoding.org/3-laws-tdd/ — Three Laws of TDD (Robert C. Martin): write no production code unless a failing test requires it
- https://martinfowler.com/articles/mocksArentStubs.html — Mocks Aren't Stubs (Fowler): the difference between state verification and behaviour verification

## Patterns & Anti-Patterns
- https://martinfowler.com/bliki/TestDouble.html — Test Doubles: stub, spy, mock, fake, dummy — when to use each
- https://martinfowler.com/articles/practical-test-pyramid.html — Practical Test Pyramid: unit, integration, E2E proportions
- https://kentbeck.github.io/desirable-tests/ — Desirable test properties: FIRST (Fast, Isolated, Repeatable, Self-validating, Timely)

## Tools
- https://docs.pytest.org/en/stable/ — pytest: Python TDD toolchain
- https://vitest.dev/ — Vitest: TypeScript/JavaScript TDD toolchain
